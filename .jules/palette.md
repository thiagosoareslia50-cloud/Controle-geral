## 2024-05-16 - Add ARIA Labels to Icon/Icon+Text Buttons for Better Accessibility
**Learning:** Found multiple buttons with icons (or just emojis) serving as actions like filtering, sorting, generating PDFs, and duplicating processes without explicit `aria-label`s or proper `aria-hidden` attributes on the icons themselves. Screen readers would read out confusing text or just the visual characters without context.
**Action:** Added `aria-label` attributes with descriptive text to `button`s and `th` sort headers. Modified the `BtnIco` wrapper component to include an `aria-hidden="true"` attribute by default so screen readers ignore decorative emojis while reading the `aria-label` or surrounding text instead. Ensure all interactive, icon-heavy elements have clear accessibility labels.
