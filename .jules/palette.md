## 2025-05-10 - Add ARIA Labels and Titles to Icon-Only Buttons
**Learning:** Icon-only buttons (using emojis like 📂, 🏢, or X symbols like ✕) without explicit labels create a severe accessibility barrier for screen reader users and a confusing experience for sighted users. The application employs conditional rendering to change the icon based on the state (e.g., toggling between "Filter by supplier" and "View all").
**Action:** Always ensure that icon-only interactive elements possess an `aria-label` attribute describing their function. Additionally, pairing the `aria-label` with a native `title` attribute guarantees that sighted users receive tooltips, creating a parity in the context provided regardless of the user's interaction modality. When icons are dynamic based on state, the `aria-label` and `title` must be dynamically updated to reflect the new action the button will perform.
