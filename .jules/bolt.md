## 2024-05-24 - Single-pass data mapping optimization
**Learning:** The `buildMapData` function in `app.js` was creating an O(n*28) performance bottleneck by executing separate `.map()`, `.forEach()`, and custom O(n) iterator functions to extract 28 different data structures (maps, sets) from the same large dataset array. This is a common anti-pattern in large buildless JS files where helper functions (`lst`, `dct`, `multi`) abstract away the fact that multiple linear scans are occurring.
**Action:** When extracting multiple indexes from a core dataset, manually unroll the abstractions into a single fast O(N) `for` loop that populates temporary `Set`s and `Map`s simultaneously. Wait to perform `.sort()` or set-to-array conversions until the single pass has completely finished processing the raw dataset.
