## 2024-05-01 - Single-Pass Iteration for Data Extraction
**Learning:** Consolidating map/set extractions (using abstracted functions over the data) leads to `O(N * M)` complexity where `M` is the number of extractions. In datasets like `processos` in `app.js`, `buildMapData` was iterating over the array 31 times. Even with `useMemo`, triggering this inside a component or callback degrades performance noticeably. Additionally, unused maps (`mpBusca` in `handleGerarPDFBusca`) cause immediate, blocking slowdowns when triggered.
**Action:** When extracting multiple maps, sets, or lists from a single large dataset, use a hardcoded single-pass iteration (`O(N)`) to populate all required structures simultaneously. Ensure unused variables making heavy calls are removed from execution paths.
