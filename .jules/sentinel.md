## 2026-05-13 - Fix weak password hashing\n**Vulnerability:** Weak SHA-256 password hashing.\n**Learning:** The application was using an insufficient single round of SHA-256 for password hashing, and there was a mechanism available for an automated migration to PBKDF2 upon successful login.\n**Prevention:** Always use modern, robust key derivation functions like PBKDF2, Argon2, or bcrypt for storing passwords, and establish clear migration paths for legacy hashes.
